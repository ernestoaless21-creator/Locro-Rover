# Locro Laravel Agent

Use this agent for feature work, bug fixes, refactors, testing, and maintenance in this repository.

## When to use this agent
- New Laravel routes, controllers, models, services, policies, or migrations
- Inertia or Jetstream UI changes in resources/js and resources/views
- Bugs affecting clients, orders, observations, payments, years, or permissions
- Requests to add tests or validate behavior before shipping

## Preferred workflow
1. Read the relevant files before editing.
2. Follow the existing structure and naming in app/, routes/, database/, and resources/js.
3. Prefer minimal, localized changes over broad rewrites.
4. Verify with the smallest relevant command, usually php artisan test or a targeted test.
5. Explain assumptions clearly and mention follow-up work if needed.

## Project context
- Laravel 12 with PHP 8.2
- Jetstream and Inertia for the UI layer
- Uses Sanctum, Spatie permissions, and Eloquent models
- Main business domain revolves around clients, orders, observations, payments, and years

## Coding expectations
- Keep controllers thin and move business logic into services or models when appropriate.
- Preserve route naming conventions and existing policy behavior.
- When changing schemas, add or update migrations and consider seeders or factories.
- Favor explicit validation and clear error handling.
- Keep changes aligned with the existing Spanish terminology and app flow.

## Tool preferences
- Prefer targeted searches and file reads first.
- Use tests and artisan commands for verification.
- Avoid unnecessary dependency additions or large refactors without discussion.

## Validation commands
- php artisan test
- php artisan route:list
- php artisan pint --test
